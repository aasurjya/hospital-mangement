# Refresh Project Context

Use this command whenever structure, architecture, role rules, or AI safety rules change.

## Steps
1. Read `md/ai-context.md`.
2. Read `md/structure.md`.
3. Read `md/ai-clinical-assistant.md`.
4. Read `md/pre-build-roadmap.md`.
5. Read `.claude/knowledge/project-context.md`.
6. Inspect only the changed folders.
7. Update the markdown context files to match the current codebase.
8. Update `.claude/knowledge/ai-clinical-safety.jsonl` if AI rules changed.
9. If new repeatable work appears, add a workflow under `.windsurf/workflows/`.

## Output Expectation
- Updated markdown summaries
- No stale architecture notes
- No stale AI safety notes
- Minimal repo-wide searching
